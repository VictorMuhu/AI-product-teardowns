# Replit Agent 4 — Text-to-Workflow Engine

Prompt for Replit Agent 4.

---

```
You are Replit Agent 4. Build a React application called the Text-to-Workflow Engine.

PRODUCT DEFINITION:
The Text-to-Workflow Engine is a decision-routing interface where users describe a workflow trigger in natural language, and the system outputs a structured routing decision — no forms, no dropdowns, no configuration UI. Input is freeform text. Output is a structured logic card.

BUILD DIRECTIVES:

1. DESIGN CANVAS — VARIANTS:
   Use the Design Canvas to generate 3 UI variants before committing to a final implementation. Variants should explore:
   - Variant A: Full-bleed dark terminal aesthetic with monospace output
   - Variant B: Clean B2B SaaS with slate palette and card-based output
   - Variant C: Minimal single-column layout with no chrome, maximum whitespace
   Select the variant with the strongest information hierarchy for the output card.

2. PARALLEL BUILD — MOBILE VIEW:
   Spin up a Parallel Build for the mobile view. The mobile layout must be treated as a first-class deliverable, not a responsive afterthought. Run the desktop and mobile builds in parallel. Resolve conflicts before merging.

3. OMNI-CHANNEL GENERATION — EXECUTIVE PITCH DECK:
   Trigger Omni-Channel generation to produce an Executive Pitch Deck from the final build. The deck must include:
   - Product vision slide (one sentence)
   - Problem / solution contrast
   - Interface screenshot (Design Canvas output)
   - Metrics slide (hypothetical, clearly labeled)
   - Call to action

STACK:
React. No UI library dependencies unless justified. Tailwind for styling. TypeScript preferred.

QUALITY GATE:
Do not submit until all three directives are complete — Design Canvas variants selected, Parallel Build resolved, and Pitch Deck generated.
```
